OVERVIEW · 四个方向

四个方向，一条主线

这四类工作最终都指向一个目标：让客户交付从"人力支持"逐步走向"产品化、可复制交付"。

DIRECTION 01

量化、导出与评估工具链

建设从量化、导出、Profile 到 HMONNX 推理加速的工程化能力，让模型适配从"能跑通"走向"可复用、可分析、可加速、可交付"。

DIRECTION 02

模型量化与芯片适配

推进 Qwen、Step3-VL、Gemma、MoE 等模型在芯片体系下可运行、可评测、可交付，覆盖通用大模型、多模态和 MoE 三类。

DIRECTION 03

下一代芯片算法定义

支撑月辉 / YH 的 reference model、C-Model、DV 与 bit-wise 对齐；将模型趋势和算法瓶颈回流到 XH3 算法需求。

DIRECTION 04

前沿算法跟进与研究

跟进低比特量化、混合精度、校准集、评估方法和服务化框架，筛选可工程化方向，输出论文 / 专利 / 算法储备。

TOPIC 01 · 量化、导出与评估工具链

工具链建设：从单点脚本走向工程化交付平台

围绕量化、导出、评估和推理加速，持续补齐 XH2 / XH2A 工具链能力，提升模型适配和客户交付的可复制性。

流程工程化

StepChain 重构、Eager 模式、ChipContext 抽象，统一不同芯片和量化流程差异。

分析可视化

Speed Profile、Precision Profile、hm-onnx-profile，支持速度、精度、算子级时延和内存分析。

推理加速

HMOps 多算子适配、CudaGraphInference、HMONNXGoldenInfer 默认加速回退机制。

算子能力补齐

LUT / SEFP GPU 加速、PReLU 特殊格式、LayerNorm / RMSNorm、flash / paged attention 浮点实现。

KEY METRICS · 突出数字
  • 4-10×HMONNX 推理加速
  • 3×SEFP CUDA 小 batch 加速
  • 1.2-1.3×激活量化 + Linear 融合算子提升

TOPIC 04 · 前沿算法跟进与研究

前沿算法研究：跟踪能力上限，筛选可工程化方向

围绕低比特量化、混合精度、校准集、评估方法和服务化框架，持续跟进算法趋势，推动有价值方向进入工具链和芯片需求。

量化算法跟进

GPTQModel、AutoRound、LightCompress、llm-compressor、AWQ / GPTQ、GGUF 等方案调研和适配。

低比特与混合精度探索

关注 4bit、5bit、W4-W8、weight-only、激活量化、混合精度、无数据量化等方向。

校准集与评估方法

EBSS、IVSE、KL Mean、PPL、ACC、Humaneval，用于量化效果和模型精度分析。

论文、专利与学术协作

DICEQuant、HIDEQuant、校准集研究、ICML 审稿 2 篇、多项专利推进。

H2 · 量化算法落地

三条算法路线并行推进

下半年模型量化工作的重点，是把可验证的量化算法真正接入工具链和评估闭环，提升模型量化效率、精度和交付稳定性。

ROUTE 01

快速 MoE 量化算法

  • 基于超球面分布角度推进
  • 面向 MoE 模型建立更高效量化策略
  • 降低调参和评估成本
ROUTE 02

Prefill / Decode 差异化量化

  • 推动 Prefill 和 Decode 使用不同量化配置
  • 分别权衡吞吐 / 延迟 / 精度 / 资源
  • 服务长上下文和真实推理场景
ROUTE 03

快速混合精度

  • 推进权重混合精度
  • 推进激活混合精度
  • 接入工具链和评估闭环
