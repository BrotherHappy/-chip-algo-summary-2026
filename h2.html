上半年暴露的四个核心问题

下半年的工作就是逐一对症。

问题域 01 · 开发流程

流程混乱、缺 review、bug 多、快速推理不够快。

问题域 02 · 量化和适配

缺标准 pipeline、缺标准评估流程和指标。

问题域 03 · 芯片需求

算法必须走在最前列，提前规避所有风险。

问题域 04 · 前沿算法

研究多但稳定落地少，需要第一性原则筛选。

02 · 四个方向行动

每个方向，都从"能支持"补齐到"标准能力"

12 件高优事项分布在四个方向 + 客户交付出口。

DIRECTION 01 · 3 件

量化、导出与评估工具链

建标准 / 通管道 / 加速度
规范化开发：增加 CI 与 Code Review，让每笔提交可溯源；标准化版本发布与依赖；发版前全自动化测试；开发文档与 Skills 整理汇总，让 Agent 也能按规范开发。
量化 + 导出 Pipeline：逻辑融合，规范接口，CLI 一键导出；Low-Resource & High-Speed，降低导出资源开销，提升导出速度，支持 200B 及以上模型导出。
评估推理加速：实现高效算子，提升整体推理速度；实现基于 vLLM 的评估，完成配套开发，作为算法落地与 XH3 验证的共同基础设施。
DIRECTION 02 · 3 件

模型量化与芯片适配

模板化 / 矩阵化 / 一键化
构建量化与适配 Pipeline：前期评估推理精度需求 / 参数量 / 算力初估 → 模型适配 → 评估 pipeline → 模型精度由高到低渐进量化 → 代码整理与发版。
推动混合精度落地：推动权重 / 激活混合精度技术，并推进编译器对应支持。
一键化 API：提供"量化、导出、评估"一键 API，可复现发版精度；建立模型族模板（Qwen / Gemma / GLM / MoE / ASR-TTS / VLA / OCR）。
DIRECTION 03 · 3 件

下一代芯片算法定义

前置评估 / 风险防范 / 深度参与
多场景模型在 XH3 上的性能前置分析与评估：覆盖 LLM / VLM / MoE / ASR-TTS / VLA 等关键场景，建立性能基线。
XH3 精度验证与风险防范：提前识别算子、shape、cache、溢出和量化敏感层风险，输出算法对硬件的依赖分析。
深入参与 XH3 整个开发流程：建立 XH3 算法需求模板（背景 / 算法要求 / 优先级 / 验收标准），形成 MoE / KV Cache / Linear Attention / FP4 / VQ / 多模态等专题需求池。
DIRECTION 04 · 2 件

前沿算法跟进与研究

实用性优先 / 真实落地
建立算法筛选标准：以"实用性"作为第一原则。三个核心问题：是否解决真实问题、是否适合我们的场景、是否能稳定落地。
推动前沿算法的真实落地：量化算法能真实落地并带来收益（FP8 / INT4 / 混合精度）；AIGC 算法做前景分析与提前适配；按 L0-L5 分级管理研究到落地的全过程。
CUSTOMER OUTLET · 1 件

客户交付共同价值出口

准入 / 交付包 / 回流
制定客户交付包模板 + 问题回流机制：明确客户准入条件与风险评估；定义交付包标准（模型 + 量化配置 + 评估报告 + 文档 + 最佳实践 + 问题处理指南）；按 P0-P3 分级响应（4h / 24h / 1 周 / 长期）。

04 · 高优事项

12 件下半年必须做的事

每件事都有明确 owner、量化标准、检查点。这是 H2 的"颗粒度抓手"。

  1. 规范化开发：CI、Code Review、版本发布、自动化测试、Skills 文档化
  2. 量化 + 导出 Pipeline：逻辑融合、CLI 一键导出、200B+ 模型支持
  3. 评估推理加速：高效算子、基于 vLLM 的评估
  4. 量化与适配 Pipeline：前期评估 / 适配 / 评估 / 渐进量化 / 发版
  5. 混合精度落地：权重与激活混合精度，编译器支持
  6. 一键化 API：量化 + 导出 + 评估，可复现发版精度
  7. XH3 性能前置分析：多场景模型性能评估与基线建立
  8. XH3 精度验证：算子 / shape / cache 风险防范
  9. XH3 深度参与：需求模板、专题需求池、三方评审机制
  10. 算法筛选标准：实用性第一，第一性原则三问
  11. 前沿算法落地：量化算法收益、AIGC 提前适配
  12. 客户交付包 + 回流：准入、交付包标准、P0-P3 SLO
05 · 节奏与检查点

前期打基础 / Q3 推进 / Q4 收口

下半年不是零散任务，而是围绕"批量交付"和"XH3 验证"两条主线展开。

JUN — JUL

基础设施先行：评估推理加速 API 初版、CI / Review 接入、量化适配 Pipeline 雏形、月辉收口阶段。

AUG — SEP

算法落地推进：MoE / Prefill-Decode / 混合精度量化进入工具链；XH3 TE 算子验证、HMOPs 复用、模型精度 align。

OCT — NOV

客户场景验证：交付包模板复现客户场景；XH3 真实模型验证、reference_model / DV 联调；问题回流到工具链与芯片需求。

DEC

阶段总结：交付流程固化、年终模板沉淀；月辉收口与 XH3 阶段总结；研究与实习生成果验收。

当这四件事都做到位，我们就不再是"能支持"的团队，而是"稳健可靠"的团队。
